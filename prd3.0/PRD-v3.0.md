## 三、首页重新设计

### 3.1 Hero 区域
- 标题: "J-Pop Hub"
- 副标题: 中文「从这里，听见日本音乐」/ 日文「ここから、日本の音楽が聴こえる」/ 英文「Discover the sound of Japan」
- 背景: 渐变圆形光晕 (#E5D89E)
- **两个 CTA 按钮**（并排）:
  - 「探索音乐人」→ 跳转 `/bands`
  - 「我的想看」→ 跳转 `/my-events`

### 3.2 移除模块
- ~~「热门推荐」Top 8 音乐人~~（v3.0 移除）

### 3.3 新增：「大事件」词云
- **数据来源**: X (Twitter) 趋势、Instagram 热门话题、TikTok 热门标签、日本 Yahoo 实时热搜、Billboard Japan 热门新闻
- **展示形式**: 词云（Word Cloud），词条大小按热度权重
- **数量**: 取 Top 10 趋势词条
- **内容示例**: Arashi 最后一场演唱会、某艺人新专辑发行、团体解散/重组、音乐节阵容公布、重大奖项结果等
- **交互**: 点击词条跳转到对应音乐人详情页或站内搜索结果
- **更新频率**: 每日刷新（静态站点通过定时构建更新，或通过客户端 API 实时拉取）
- **过滤**: 仅展示与日本音乐相关的词条，去除非音乐类泛热搜

### 3.4 新增：「即将截止购票」
- **展示形式**: 横向卡片排布（3 张）
- **数据来源**: 从所有音乐人的巡演信息中筛选，取售票截止日期最近的 3 场
- **排序**: 按截止日期紧急程度（最近截止优先）
- **每张卡片展示**:
  - 音乐人头像（小图）
  - 音乐人名称
  - 巡演名称
  - 售票截止日期 + **倒计时**（如「3 天后截止」）
  - 购票链接按钮
- **无数据时**: 不展示该模块，或显示「暂无即将截止的售票」

### 4.5 音乐人详情页 — 巡演「想看」按钮 🆕

在音乐人详情页的**巡演信息（Tours）**模块中，每条巡演卡片新增「想看」按钮。

#### 4.5.1 按钮设计
- 位置：每条巡演卡片的右下角（与「购票链接」按钮并列或在其旁边）
- 图标：❤️ / ⭐ 心形或星形图标 + 文字「想看」
- 两种状态：
  - **未收藏**：空心图标 + 「想看」文字
  - **已收藏**：实心图标 + 「已想看」文字（高亮色）

#### 4.5.2 点击行为
- 点击「想看」→ 该巡演信息自动保存到 `localStorage` 中的 `wantToGoTours` 数组
- 保存内容：音乐人名称、巡演名称、日期、地点、购票链接
- 按钮状态切换为「已想看」（实心图标）
- **无需登录**，数据存储在浏览器本地

#### 4.5.3 数据流向
```
音乐人详情页巡演卡片 [点击「想看」]
        ↓
存入 localStorage.wantToGoTours[]
        ↓
/my-events 页面读取并展示
```

#### 4.5.4 技术实现
- 纯前端逻辑，不需要后端 API
- 使用 React Context 或 zustand 管理「想看」状态
- 页面加载时从 localStorage 读取已收藏的巡演列表，匹配当前页面的巡演 ID 来显示按钮状态

## 六、个人主页 (`/my-events`) 完全重构

### 6.1 用户数据方案（纯前端 localStorage）
- **无需注册/登录**：用户直接访问 `/my-events` 即可使用
- **数据存储**: 所有用户数据存储在浏览器 `localStorage` 中
- **数据结构**:
  ```typescript
  {
    wantToGoTours: Tour[],      // 想看的巡演（来自详情页点击 + 自行添加）
    followedArtists: string[],  // 关注的音乐人 slug 列表
    todoItems: TodoItem[],      // To Do 事项
  }
  ```
- **局限性**: 清除浏览器数据会导致数据丢失，不支持跨设备同步
- **未来升级方向**: 接入 CloudBase 等 BaaS 实现云端存储和跨设备同步

### 6.3 模组一：「想看的巡演」

#### 6.3.1 搜索已有巡演
- 搜索框：输入音乐人名或巡演名称
- 数据源：从「音乐人信息」板块中所有**未来日期**的巡演信息中筛选
- 搜索结果展示：音乐人名称、巡演名称、日期、地点
- 点击「添加」即可收藏到自己的列表

#### 6.3.2 自行添加巡演
- 弹窗/表单，填写字段：
  - 日期（必填）
  - 演出者（必填）
  - 地点/场馆（选填）
  - 购票链接（选填）
  - 备注（选填）
- 提交后加入个人巡演列表

#### 6.3.3 巡演列表展示
- 按日期排序（最近的在前）
- 每项显示：日期、演出者、地点、购票链接、备注
- 支持删除/编辑
- 分为「即将到来」和「已过期」两个子列表

#### 6.3.4 日元/人民币汇率
- 模块底部固定显示
- 格式：`💱 今日汇率: 1 JPY ≈ 0.0XXX CNY（更新于 YYYY/MM/DD）`
- 数据来源：调用免费汇率 API（如 exchangerate-api.com）
- 缓存 24 小时

### 6.4 模组二：「关注的音乐人」

#### 6.4.1 添加方式
- 搜索选择：从 50 位已有音乐人中搜索并添加
- 手动输入：添加网站未收录的音乐人名称（仅名称，无自动数据聚合）

#### 6.4.2 列表展示
- List 形式，每项展示：
  - 音乐人头像 + 名称
  - **最新社交动态**（取最近 1-3 条，来自 X / IG / TikTok / YouTube）
  - **最新巡演信息**（最近一场未来日期的巡演）
  - 动态折叠/展开
- 按最新动态时间排序
- 支持取消关注

## 七、大事件词云 — 技术方案

### 7.1 数据聚合
- 静态站点方案：通过 Node.js 脚本定时抓取，将结果写入 JSON 文件，随站点一起构建
- 动态方案（推荐）：客户端通过 API 拉取实时热搜数据
- 数据源：
  - X (Twitter) Trends API / 第三方趋势接口
  - Instagram 热门标签
  - TikTok Trending Hashtags
  - Yahoo Japan 实时热搜
  - Billboard Japan Hot 100 / News

### 7.2 词云渲染
- 使用词云库（如 `react-wordcloud` 或自建 Canvas 实现）
- 词条大小按热度值映射
- 颜色使用品牌色系（橄榄绿、米黄、深绿等渐变）
- 点击词条跳转逻辑：
  - 若匹配到已有音乐人 → 跳转该音乐人详情页
  - 否则 → 跳转站内搜索页（以词条为搜索关键词）

### 7.3 过滤机制
- 仅展示与日本音乐相关的词条
- 黑名单：过滤政治、社会新闻等无关词条
- 白名单/关键词匹配：包含音乐人名称、音乐术语、演出相关词汇

## 十、技术架构

### 10.1 部署约束
v3.0 **必须保持 GitHub Pages 纯静态部署**，不支持 Vercel 或其他需要服务端的平台。
因此所有功能必须在静态站点的约束下实现：

| 方面 | 实现方式 |
|------|----------|
| 部署方式 | **GitHub Pages (纯静态 SSG)** |
| 用户数据 | **localStorage**（客户端持久化，无需后端） |
| 用户系统 | 无后端注册/登录，使用 localStorage 存储用户偏好；「我的想看」等数据存储在浏览器本地 |
| 词云数据 | 通过 Node.js 脚本**定时抓取并生成 JSON 文件**，随站点一起 SSG 构建 |
| 汇率数据 | 通过 Node.js 脚本**定时抓取并写入静态 JSON**，随站点构建；或客户端 fetch 免费 API |
| 巡演「想看」 | 点击后将巡演信息存入 localStorage，在 `/my-events` 页面读取展示 |

### 10.2 用户数据方案（纯前端）
- **不需要注册/登录**：用户直接使用「我的想看」功能
- 数据存储在浏览器 `localStorage` 中
- 数据结构：
  ```typescript
  {
    wantToGoTours: Tour[],      // 想看的巡演
    followedArtists: string[],  // 关注的音乐人 slug 列表
    todoItems: TodoItem[],      // To Do 事项
  }
  ```
- **局限**：清除浏览器数据会导致数据丢失，不支持跨设备同步
- 未来可升级：接入 CloudBase 等 BaaS 实现云端存储

### 10.3 定时数据更新方案
- 使用 GitHub Actions 定时运行 Node.js 脚本
- 脚本抓取热搜词条、汇率数据，生成 JSON 文件
- 自动 commit 并 push 到仓库，触发 GitHub Pages 重新部署
- 频率：每日一次

### 10.4 词云渲染
- 使用 `react-wordcloud` 或 D3-cloud（纯前端库）
- 数据来自预生成的静态 JSON 文件
- 点击词条跳转：匹配音乐人 → 详情页；否则 → 搜索页
